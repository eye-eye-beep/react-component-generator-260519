# TDD Rule — React Component Generator

⚠️ **이 규칙은 Rigid** — 상황에 맞게 변형하지 마라. 프로젝트 CLAUDE.md/AGENTS.md에 TDD 규칙이 있으면 그것을 우선한다.

## 1. 적용 기준 구분

### TDD를 반드시 적용하는 대상

- **비즈니스 로직**: 상태 관리 (`useComponentGenerator`), 컴포넌트 생성 로직, API 호출 흐름
- **유틸 함수**: `stripCodeFences`, `ensureRenderCall`, 코드 후처리, 유효성 검증
- **API 라우트**: 서버 엔드포인트 (`/api/generate`, `/api/config`), 요청/응답 처리
- **버그 수정**: 기존 버그의 원인을 테스트로 재현 후 수정 (회귀 방지)
- **훅 로직**: 복잡한 커스텀 훅의 상태 전환, 부작용 시퀀싱

### TDD가 불필요한 대상

- **타입 정의** (`src/types/index.ts`): TypeScript 컴파일러가 검증
- **설정 파일** (`.env`, `vite.config.ts`, `tsconfig.json`)
- **순수 UI 컴포넌트**: 렌더링만 담당하는 프레젠테이션 컴포넌트 (`ComponentCard`, `CodeView` 등)
- **react-live 프리뷰**: 런타임 렌더링은 수동 테스트 (브라우저)
- **외부 라이브러리 래핑**: 써드파티 API 사용 (`react-live`, Anthropic SDK, Google Gemini)
- **스타일 / CSS**: 인라인 스타일, 레이아웃

## 2. RED-GREEN-REFACTOR 사이클

### RED: 실패하는 테스트 작성

1. **한 가지 동작만 테스트**하는 테스트를 먼저 작성 — 각각을 **개별 테스트**로 작성

2. **반드시 실행해서 FAIL 확인**
   - `bun run test`
   - 실패 이유가 **"기능 미구현"**이어야 함 (구문 오류, 모듈 없음 ≠ 실패)

3. **테스트 코드만 완성** — 구현은 한 글자도 금지

### GREEN: 최소 코드로 테스트 통과

1. **YAGNI** — 테스트를 통과하는 **최소한의 코드만** 작성, "나중에 쓸 것 같은" 기능 추가 금지

2. **신규 + 기존 테스트 모두 통과 확인**
   ```bash
   bun run test
   ```

3. 간단한 하드코드 허용 (GREEN만 목표, REFACTOR에서 개선)

### REFACTOR: 중복 제거 & 개선 (GREEN 유지)

**할 수 있는 것:**
- 중복 코드 추출 → 헬퍼 함수화
- 변수/함수명 개선
- 불필요한 임시 변수 제거

**금지 사항:**
- **새로운 동작 추가 금지** (다시 RED로)
- 구조 전혀 다르게 바꾸기 금지 (REFACTOR ≠ 재설계)
- 테스트 코드 수정 금지

REFACTOR 후 반드시 `bun run test` 실행 — 모든 테스트 통과 확인

### 사이클 반복

RED → GREEN → REFACTOR → **다음 동작의 RED** → 반복

## 3. 삭제 강제 규칙

**테스트 없이 먼저 코드를 쓴 경우:**
1. **작성한 구현 코드 모두 삭제**
2. **RED부터 다시 시작** — 테스트 먼저 작성 → 실패 확인 → 구현

**"참고용으로 남겨두겠습니다" 금지:**
- 주석 처리된 코드도 제거
- `// TODO: 나중에 구현` 플레이스홀더 금지
- 백업은 git 브랜치로 관리

## 4. 변명 차단표

| 변명 | 반론 |
|------|------|
| "너무 단순해서 테스트 불필요" | 단순한 것이 테스트하기 쉽다. 가장 먼저 테스트할 대상이 단순한 함수들이다. |
| "나중에 테스트 추가하겠다" | 나중은 오지 않는다. 지금 RED를 작성하면 같은 시간에 기능이 두 배다. |
| "시간이 없다" | TDD는 **시간을 버린다**가 아니라 **시간을 빌려간다**. 디버깅 시간을 아낀다. |
| "이 코드 삭제하면 낭비" | 이미 낭비한 시간이다. RED부터 다시 시작해야 진짜 가치를 안다. |
| "프로토타입이니까 TDD 스킵" | 프로토타입이 프로덕션이 된다. 지금부터 테스트 습관을 들여야 한다. |
